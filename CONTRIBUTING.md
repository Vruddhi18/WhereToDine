# Contributing to WhereToDine

First off, thank you for considering contributing to WhereToDine! It's people like you that make WhereToDine such a great tool for restaurant recommendations.

We welcome all types of contributions, including but not limited to:
- Bug reports and fixes
- Feature requests and implementations (new recommendation strategies, UI improvements, etc.)
- Documentation improvements

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/WhereToDine.git
   cd WhereToDine
   ```
3. **Set up the local environment:**
   Follow the Installation and Usage instructions in the [README.md](README.md) to set up the FastAPI machine learning server and the Node.js authentication server.

## Making Changes

1. **Create a new branch** for your feature or bugfix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
   Or for a bug:
   ```bash
   git checkout -b fix/your-bugfix-name
   ```
2. **Make your changes** in your local branch. Ensure your code follows the existing style, and comment complex logic (especially in the recommendation algorithms).
3. **Write clear commit messages**:
   - Use the present tense ("Add feature" not "Added feature").
   - Keep the first line concise.
   - Reference any related issues or pull requests.

## Submitting Your Changes

1. **Push your branch** to your fork on GitHub:
   ```bash
   git push origin feature/your-feature-name
   ```
2. **Open a Pull Request (PR)**:
   - Go to the main [WhereToDine repository](https://github.com/Vruddhi18/WhereToDine).
   - Click the "Compare & pull request" button.
   - Describe your changes in detail in the PR description, explaining *what* was changed and *why*.
   
## Code of Conduct

By participating in this project, you agree to abide by healthy open source etiquette. Be respectful to others, welcoming to beginners, and constructive in your feedback and code reviews.

Thank you for contributing!
